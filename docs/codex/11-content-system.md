# Content System

## Purpose

The Content System centralizes the information Harmonic OS displays so the public website, Control Hub, Investor Mode, and analytics can all use the same source of truth.

## Content Categories

### Worlds

Each world should have:

- Name.
- Description.
- Tagline.
- Visual mood.
- Primary CTA.
- Secondary CTAs.
- Social links.
- Featured content.

### Social Links

Socials should be stored once and reused.

Groups:

- Melodic.
- 2 Harmonic.
- Fried Em.
- Schmackinn.
- Support.

### Videos

Fields:

- Title.
- Platform.
- URL.
- World.
- Thumbnail.
- Description.
- Featured status.

### Music

Fields:

- Song title.
- URL.
- Platform.
- Mood.
- Featured status.

### Clothing Drops

Fields:

- Product/drop name.
- Season.
- Colors.
- Description.
- Images.
- Shop URL.
- Release status.

### Announcements

Fields:

- Title.
- Message.
- Start date.
- End date.
- CTA.
- Active status.

## Build Strategy

v0.1 can use static data files.

Later versions connect data to Harmony Core so the owner can edit without coding.
